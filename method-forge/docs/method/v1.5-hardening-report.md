## 3. 设计意图

这批增强保持了 `method-forge` 的方法层边界：

- `code-review` 是审查方法，不是新 review 平台
- `memory-promote` 是候选整理方法，不是 memory system 改造
- lint 与 health check 是治理守则，不是强制执行引擎

## 4. 新的推荐扩展流

高风险任务可走：

```text
implement
-> code-review
-> verify-change
-> memory-promote
```

其中：

- `code-review` 可选，但高风险改动推荐启用
- `memory-promote` 只在 `verify.md` 已认定 candidate 合格时才运行

## 5. 仍然不做的事

- 不直接写 memory
- 不复制 Codex 原生 diff review
- 不引入后台 orchestrator
- 不做强制模板编译器或完整 lint 工具链
